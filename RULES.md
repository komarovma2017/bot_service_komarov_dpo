# RULES.md — Правила разработки проекта

## Цель этого файла
Claude Code CLI читает этот файл перед каждой задачей.
Здесь описаны правила, которые НЕЛЬЗЯ нарушать при генерации кода.

---

## 1. Язык и стиль кода
- Язык кода: Python 3.11+
- Все комментарии в коде: **на русском языке** (учебный проект)
- Стиль: PEP8, форматирование через `ruff`
- Имена переменных/функций/классов: **на английском языке**
- Строки документации (docstring): на русском языке

## 2. Слои архитектуры (строго соблюдать порядок)
```
HTTP Request → Route (api/) → Usecase (usecases/) → Repository (repositories/) → DB
```
- **Route** — только принимает запрос, вызывает usecase, возвращает ответ. Никакого SQL.
- **Usecase** — бизнес-логика. Вызывает репозиторий. Бросает HTTPException-наследников.
- **Repository** — только SQL/ORM операции. Не бросает HTTPException.
- **Core** — конфиг, безопасность, исключения. Не зависит от HTTP.

## 3. Обработка исключений
- Создавай кастомные исключения в `app/core/exceptions.py`
- Наследуй от `BaseHTTPException(HTTPException)`
- В usecases НИКОГДА не пиши `raise HTTPException(...)` напрямую
- Используй: `UserAlreadyExistsError`, `InvalidCredentialsError`, `InvalidTokenError` и т.д.

## 4. JWT правила
- **Auth Service**: создаёт JWT с полями `sub`, `role`, `iat`, `exp`
- **Bot Service**: ТОЛЬКО декодирует и валидирует JWT. Никогда не создаёт.
- Алгоритм: HS256 (учебный вариант, общий секрет)
- Секрет: берётся из переменной окружения `JWT_SECRET`

## 5. База данных (Auth Service)
- Используй `SQLAlchemy async` + `aiosqlite`
- `Base = DeclarativeBase()` только в `app/db/base.py`
- Сессия через `Depends(get_db)` в deps.py
- Пароль хранить ТОЛЬКО как хеш (bcrypt через passlib)
- Уникальный индекс на поле `email` в модели User

## 6. Celery и очереди
- Брокер: RabbitMQ (RABBITMQ_URL из env)
- Backend результатов: Redis (REDIS_URL из env)
- Задача LLM: `llm_request` в `app/tasks/llm_tasks.py`
- В Telegram-хэндлере: только `llm_request.delay(tg_chat_id, prompt)`
- НИКОГДА не вызывать OpenRouter API прямо в хэндлере

## 7. Redis в Bot Service
- Клиент через `get_redis()` из `app/infra/redis.py`
- Ключ хранения токена: `token:<telegram_user_id>`
- Не создавать новый клиент Redis при каждом вызове

## 8. Тесты
- Auth Service: unit-тесты (security functions), интеграционные (HTTP endpoints)
- Bot Service: unit-тесты (JWT decode), мок-тесты (handlers с fakeredis)
- Интеграционные тесты OpenRouter: через `respx` (без реальных HTTP запросов)
- В тестах НИКОГДА не обращаться к реальным Redis/RabbitMQ
- Используй `fakeredis` и патчи через `pytest-mock`

## 9. Docker
- docker-compose включает: auth_service, bot_service, celery_worker, rabbitmq, redis
- Имена сервисов в docker-compose: `rabbitmq`, `redis` (именно так настроены .env файлы)
- Каждый сервис имеет healthcheck

## 10. Что НЕ должно быть в коде
- Никаких упоминаний AI-инструментов (Claude, ChatGPT и т.д.)
- Никаких `# Generated by Claude` или подобных комментариев
- Никакого кода, нарушающего разделение слоёв
- Никаких синхронных операций с БД (только async)
